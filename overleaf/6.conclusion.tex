\section{Limitations}
\label{sec:limitation}
The proposed passive anxiety tracking system achieved a 69.2\% average accuracy through its SVM-RBF kernel but the engineering assessment revealed various structural and environmental and analytical limitations which need to be recognized. The engineering assessment has identified various structural and environmental and analytical boundaries which demand recognition although the passive anxiety tracking system achieved a mathematically significant average accuracy of 69.2\% through its SVM-RBF kernel implementation:

\begin{itemize}
    \item \textbf{Limited Cohort Demographics and Ecological Constraints:} The telemetry data which served as training material for machine learning classifiers during their optimization and cross-validation process came from an experimental group of 67 first-year computer science students who took their exams under active timed terminal examination conditions. The environment delivered outstanding internal validity because it created authentic psychological stress which produced actual stress reactions yet the setup established particular limits. The professional software engineers who work in enterprise environments show distinct coding patterns and stress management abilities and work schedules and stress handling techniques which create challenges for using this model in real-world situations outside academic testing environments.

    \item \textbf{Platform Dependency and Decoupled Constraints:} The architectural data logging loop operates with strong desktop environment dependence because it operates best when it records application window changes and keyboard activities in the Code::Blocks editor and Windows command shells. The approach runs smoothly at the operating system level because it avoids any issues with different versions of asynchronous monitoring services but it blocks users from performing immediate cross-platform operations. Modern engineering teams need abstracted tracking systems because they operate multiple cloud-based and web-based code editors which include Visual Studio Code and JetBrains IntelliJ IDEA and containerized cloud IDEs. The tracking systems must support multiple operating system window managers because these environments demand developers to create adaptable tracking systems.

    \item \textbf{Subjective Evaluation Bias in Ground Truth Labeling:} The machine learning models employ binary classification targets which depend on user stress data that individuals report through their post-exam 5-point Likert scale responses. Human beings evaluate their own emotions through a personal process which depends on how they performed recently at work and their individual characteristics and their current level of exhaustion. The ground truth labels contain natural statistical noise which creates an unbreakable barrier for classifier performance that no algorithmic optimization can surpass.

    \item \textbf{Absence of Biological and Physiological Signals:} The classification system restricts its data stream access to passive mechanical software interaction telemetry which includes information about how users type and their window switching behavior and delete command usage. The system protects user privacy through its hardware-free design because users need to spend no money for implementation but it does not provide any method to verify biological data. The framework fails to measure physiological stress responses because it does not collect data about Autonomic Nervous System activation through Galvanic Skin Response (GSR) and Heart Rate Variability (HRV) which would offer an objective biological marker to support behavioral signs.

    \item \textbf{Binary Classification Simplification:} The machine learning system operates through a binary classification system which identifies either High Stress or Low/Moderate States to achieve maximum statistical power together with unambiguous prediction results. The initial data gathering process along with the rule-based heuristic system operated on a detailed four-level scale which included LOW, MODERATE, HIGH, and CRITICAL to describe the data. The predictive model uses a binary threshold which accepts Likert survey scores at level 4 or higher to address the problem of insufficient data points and unbalanced class distribution. The system operates through a binary structure which enables it to detect high-stress states by producing simple results that academic assessment monitoring systems can use for their decision-making processes.
\end{itemize}

\section{Summary}
A zero-intrusion developer anxiety monitoring system which operates without hardware was developed in this thesis through field testing of real-world data. The study examined more than 11,000 individual 30-second observation periods which researchers recorded from 67 students who took actual exams to establish that operating system features which operate in the background directly affect human stress levels ($p < 0.01$). The research shows that a Support Vector Machine (SVM-RBF) classifier achieves 69.2\% accuracy in identifying coding stress states through a 10×10 Repeated Stratified Cross-Validation loop which works with SMOTE dataset balancing. The system operates at the top of the competitive range because it achieves 69.2\% accuracy in subject-independent universal model classification. The multi-state high-fidelity interactive prototype serves as a working model which shows how to build future context-aware interfaces that operate without needing user intervention. The two contributions solve a vital problem which exists between computer science education and developer tool creation through their development of a scalable framework which runs without any operational costs for emotional intelligence systems.
