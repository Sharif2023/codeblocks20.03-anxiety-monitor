\chapter{Implementation and Results}

This chapter accounts for the complete technical integration and empirical evaluation of the real-time anxiety detection system. It details the runtime architecture operating asynchronously at the operating system level, transitions into the automated data extraction and cleaning pipeline deployed during active university examinations, provides a statistical correlation validation of the raw tracking indices, and evaluates binary stress classification performance across multiple machine learning models with synthetic dataset balancing.

\section{System Development and Environment Setup}
The engineering objective of this study focuses on tracking and classifying developer anxiety dynamically without introducing intrusive hardware or artificial testing protocols that compromise the ecological validity of the coding environment. Consequently, the telemetry logging architecture was designed as a lightweight background monitor optimized for C/C++ programming workflows, with specific contextual tuning for the Code::Blocks editor workspace.

\subsection{Runtime Architecture}
To eliminate dependency vulnerabilities, reduce processing footprint, and ensure hidden execution, the background monitor bypasses traditional IDE-specific telemetry hook frameworks. Instead, it utilizes a decoupled runtime architecture implemented directly at the operating system (OS) API layer. The tracker runs asynchronously as a background system service, intercepting high-frequency user interactions without impacting compiler thread priorities or adding execution latency to the developer’s workspace. 

\noindent The logging mechanism continuously hooks into the native OS keyboard event stream and active window foreground descriptors. It actively maps the exact binary image currently occupying user focus, tracking context-switching transitions among the core IDE editor core (\texttt{codeblocks.exe}), the underlying compiler/debugger binaries (\texttt{gcc.exe}, \texttt{gdb.exe}), and the runtime console environment (\texttt{cmd.exe}). Aggregated features are calculated and serialized onto a consistent 30-second tumbling window sequence, preserving high ecological validity while maintaining zero performance overhead on the primary development machine.

\subsection{Rule-Based Anxiety Score Engine}
Prior to deploying advanced supervised classification algorithms, an expert-driven heuristic system was constructed to establish an analytical baseline using established Human-Computer Interaction (HCI) principles. This internal engine maps known behavioral distress indicators specifically computing a proportional relationship between erratic typing speed fluctuations, prolonged pauses, and elevated deletion rates to output a baseline linear mathematical value:

\begin{equation}
\text{Anxiety\_Score} = f(\text{Latency\_Variance}, \text{Backspace\_Rate}, \text{Pause\_Ratio})
\end{equation}

\noindent The rule-based calculation normalizes feature inputs into an \texttt{anxiety\_score} ranging bounded from 0 to 100. This score is split into four explicit categorical stress domains: \texttt{LOW}, \texttt{MODERATE}, \texttt{HIGH}, and \texttt{CRITICAL}. Although this static rule layer lacks the dynamic thresholding flexibility of deep classifiers, it yields the underlying continuous metric required for statistical baseline validation against self-reported psychological instruments.

\subsection{Study Context and Participant Demographics}
To gather authentic psychological stress responses rather than simulated conditions, data collection was conducted during live, graded undergraduate academic examinations. The experimental cohort comprised 67 first-year computer science and engineering students, all aged below 25, who completed complex algorithmic challenges under strict, timed assessment constraints. Because performance directly impacted course marks, the examination context induced high physiological and psychological stress responses, guaranteeing strong data integrity and ecological validity.

\subsection{Data Cleaning and Preprocessing Pipeline}
Because the raw data stream was collected in unconstrained testing environments, it contained noise artifacts from mechanical disruptions or long textual reading intervals. A structured three-stage data cleaning pipeline was implemented to condition the feature matrix before model training:

\begin{enumerate}
    \item \textbf{Warm-up Filtering}: Initial time-series rows capturing an empty keyboard buffer (\texttt{keystrokes\_total} $= 0$) were systematically purged. This filters out the initial reading phase when a student silently reviews the exam description, preventing empty typing intervals from artificially deflating the true average typing speed profile.
    \item \textbf{Outlier Capping}: Mechanical and behavioral outliers were bounded using hard mathematical constraints. For instance, prolonged multi-minute pauses caused by an instructor clarifying a question would cause the typing hesitation variance metric to expand exponentially. To stop these extreme intervals from heavily distorting the statistical variance calculations, \texttt{latency\_variance\_ms} was capped at a strict maximum limit of 30,000 milliseconds.
    \item \textbf{Data Merging and Aggregation}: The separated log tables from all 67 participants across the four distinct exam sessions were computationally concatenated and structured. This merging stage yielded a standardized, unified time-series dataset containing exactly 11,138 distinct 30-second observation windows for feature engineering.
\end{enumerate}

\subsection{Exploratory Data Analysis (EDA) and Label Distribution}
To build a dependable target framework for supervised machine learning, objective behavioral features were paired with a recognized ground truth label. Within 5 to 15 minutes post-examination ensuring clear psychological recall participants completed a digital assessment mapping their perceived stress on a 5-point Likert scale (where 1 indicates low stress and 5 represents severe stress). 

\noindent The resulting label distribution exhibited a heavy natural class imbalance, with approximately 83\% of participants scoring within the high stress bands (levels 4 and 5), matching the emotional baseline of highly pressured academic assessments.

\section{Descriptive Behavioral Feature Profiling}
To expose the specific behavioral micro-signatures captured during testing, a descriptive statistical extraction was conducted on the compiled time-series interaction logs. Segmenting the fine-grained telemetry data across the verified model target boundaries highlights clear differences in user input geometries as developers experience escalating cognitive load. 

\noindent When a programmer operates within the \texttt{LOW} anxiety classification band, their physical interaction parameters reflect a highly rhythmic, stable, and unimpeded coding flow. Within this baseline, typing velocity maintains a steady mean of $6.13$ words per minute (WPM), while the primary indicator of typing hesitation \texttt{latency\_variance\_ms} remains exceptionally flat at an average of only $957.14$ milliseconds. This rhythmic consistency is supported by minimal logical corrections, with the backspace rate percentage averaging a low $1.60\%$. Furthermore, environmental interactions indicate highly focused work: developers show an average of only $2.2$ application focus switches per 30-second window, remaining firmly inside the \texttt{codeblocks.exe} text canvas.

\noindent As cognitive friction or structural blockages develop, the mechanical signature changes distinctly within the \texttt{MODERATE} anxiety classification band. Here, the programmer's typing velocity drops to a mean of $2.65$ WPM, signaling immediate procedural hesitation. Concurrently, \texttt{latency\_variance\_ms} steps up to an average of $9,568.92$ milliseconds, highlighting irregular intervals between keypresses. Frustration markers become apparent as the backspace rate percentage shoots up to $11.33\%$, reflecting repetitive deletion and syntax modification loops. Spatial fragmentation also increases, with application window focus switching jumping to a mean of $64.39$ shifts per interval as developers begin bouncing between the source code file, background specifications, and local help resources.

\noindent Under severe cognitive pressure, the system logs chaotic interaction parameters within the \texttt{HIGH} and \texttt{CRITICAL} anxiety classification bands. Typing velocity slumps to its lowest operational mean of $2.37$ WPM, accompanied by an absolute plateau in the text buffer. Crucially, \texttt{latency\_variance\_ms} spikes to an extreme average of $24,205.79$ milliseconds, capturing a pattern of long, frozen stalls followed by frantic bursts of typing as the exam timer runs out. Deletion rates remain elevated at a mean backspace rate of $10.59\%$, while the focus-switching metric accelerates to a substantial mean of $164.01$ window toggles per interval. This extreme level of environment toggling reflects total session fragmentation, indicating that high anxiety limits working memory capacity and forces developers to constantly switch focus between windows to re-verify syntax errors and compile logs.

\section{Validation Against Self-Reported Stress}
Before training machine learning models, statistical correlation tests were run to confirm the presence of reliable psychological stress markers within the passive logging telemetry. The localized continuous index from the heuristic rule engine was validated against the users' self-reported Likert ground truth labels \cite{likert1932}.

\noindent By flattening the 11,138 intervals into average behavioral matrices for each of the 67 students, two formal correlation tests were calculated:
\begin{itemize}
    \item \textbf{Pearson Correlation (Linear Trend Evaluation)}: $r = 0.521$ ($p < 0.0001$)
    \item \textbf{Spearman Rank Correlation (Monotonic Trend Evaluation)}: $\rho = 0.310$ ($p = 0.0107$)
\end{itemize}

\noindent Both evaluations returned statistically significant results well below the standard behavioral alpha threshold ($p \ll 0.05$), confirming a reliable relationship between passive mechanical key inputs and the programmer's underlying emotional stress state.

\section{Machine Learning Evaluation}

\subsection{Task Definition and Protocol}
Once the presence of a statistical signal in the behavioral data was verified, the fixed linear heuristic formula was replaced by supervised machine learning models to enable automated classification. The analytical objective was defined as a Binary Classification task: predicting whether a given 30-second window represents a high stress state (Likert survey target $\ge 4$) or a low-to-moderate baseline state (Likert survey target $< 4$) using only passive keyboard and window metrics.

\noindent Because the dataset suffered from a heavy majority-class distribution (~83\% high stress records), standard unmitigated classifiers would become biased, achieving a false accuracy baseline of 83\% simply by predicting the majority class label exclusively. To overcome this class imbalance trap, the training framework incorporated two strict preprocessing and validation safeguards:
\begin{itemize}
    \item \textbf{SMOTE Balancing}: The Synthetic Minority Over-sampling Technique \cite{chawla2002smote} was applied to interpolate new data nodes within the minority feature coordinates (low stress), establishing a mathematically balanced, 50/50 dataset for model training. SMOTE was applied using the \texttt{imbalanced-learn} library integrated into the Scikit-learn pipeline \cite{scikitlearn}.
    \item \textbf{10$\times$10 Repeated Stratified K-Fold Cross-Validation}: To eliminate evaluation bias, cross-validation split the data over 100 separate evaluation loops. Crucially, SMOTE was executed strictly inside the training split folds of each fold loop, preventing data leakage from the validation test splits into the learning phase \cite{smoteLeakage}.
\end{itemize}

\subsection{Model Performance Analysis}
Following SMOTE balancing, the random guessing baseline dropped to exactly 50\%, meaning any classification accuracy significantly above this score validates the model's capacity to recognize stress patterns from mechanical typing data.

\noindent Evaluating the balanced dataset across the 100 cross-validation training cycles highlighted three top-performing machine learning architectures:
\begin{enumerate}
    \item \textbf{Support Vector Machine (SVM with RBF Kernel)}: $69.2\% \pm 12.9\%$
    \item \textbf{Naive Bayes Classifier}: $68.0\% \pm 16.2\%$
    \item \textbf{Random Forest Ensemble}: $66.4\% \pm 16.0\%$
\end{enumerate}

\noindent Achieving a Mean Accuracy of 69.2\% against the 50\% baseline validates the feasibility of passive stress tracking. The Support Vector Machine using a Radial Basis Function (RBF) kernel proved the most robust at drawing non-linear hyperplanes to separate the complex interaction telemetry.

\section{Results and Discussion}

\subsection{Behavioral Signatures of Coding Stress}
The feature importance rankings and continuous statistical correlations outline clear behavioral signatures of development stress. Crucially, the predictive weight of \texttt{latency\_variance\_ms} far exceeds that of raw \texttt{typing\_speed\_wpm}. This reveals that a developer experiencing stress does not simply type at a universally faster or slower speed; rather, their input rhythm becomes highly \textit{irregular}. High anxiety triggers cognitive hesitation loops, causing long periods of typing inactivity while a student struggles to resolve logical blockages, followed by rapid, frantic bursts of typing as the assessment deadline nears.

\noindent Additionally, the increased backspace rate caught during high-stress periods matches historical psychological findings on development frustration. First-year students adapting to complex programming architectures often lack debugging confidence under pressure, making them prone to syntax errors that trigger repetitive deletion and rewriting behaviors.

\subsection{Contextual Influence of IDE Activity Phases}
Furthermore, variations in \texttt{focus\_switches} and \texttt{compile\_success\_rate} illuminate how stress alters a student's spatial interaction with their digital environment. Stressed students show elevated window focus switching, rapidly bouncing focus between the source code file (\texttt{codeblocks.exe}), compiler error outputs, and background requirements sheets.

\noindent This behavioral pattern, termed "Session Fragmentation," indicates that elevated cognitive load compromises a developer's working memory capacity. Overwhelmed students struggle to maintain contextual problem-solving focus in a single window, forcing them to repeatedly shift focus across applications to re-verify syntax specs or error strings. These clear window context switches validate the effectiveness of tracking OS-level app interactions as non-intrusive indicators of psychological load.

\subsection{Performance of the Predictive Models}
The supervised learning pipeline proves that models can successfully classify a programmer's distress state with a mean accuracy of 69.2\% using only passive keyboard and window metadata  operating entirely without wearable sensors, physiological probes, or IDE instrumentation. While traditional clinical research relies on intrusive biomedical devices such as multi-channel EEG caps, chest-strap heart rate monitors, or galvanic skin response wristbands to isolate emotional spikes, this framework achieves statistically significant predictive power with zero physical contact or setup overhead on the student machine. For real-world educational labs or enterprise workplace monitoring where physical biometric sensors are impractical to deploy at scale, this passive OS-level telemetry architecture offers a robust and immediately scalable alternative.

\subsection{Comparison with Related Work}
To contextualize the performance of this system within the broader body of behavioral biometrics and HCI stress detection research, it is critical to distinguish between two fundamentally different model paradigms that exist in the literature.

\noindent The first paradigm is the \textbf{Subject-Dependent (Personalized) Model}, in which a dedicated classifier is trained and calibrated on an individual user's own historical baseline typing data before being deployed for that specific person. Because these models learn the unique rhythmic fingerprint of one typist, they can detect deviations from that individual's personal baseline with high precision. Studies following this personalized paradigm commonly report classification accuracies in the range of \textbf{75\% to over 90\%}~\cite{cmuKeystroke}. However, this architecture carries a critical practical limitation: a new user cannot be monitored until a substantial individual calibration period has been completed, making it unsuitable for one-time or first-visit deployment scenarios such as an academic examination.

\noindent The second paradigm is the \textbf{Subject-Independent (Universal) Model}, in which a single classifier is trained across a large, diverse cohort of users and must generalize its stress predictions to entirely unseen individuals without any prior individual calibration. This is the paradigm adopted by the present study. Because human typing behaviors are highly individualized  what constitutes a ``fast'' typing speed for one person may represent a ``slow'' rate for another  universal models face a fundamentally harder classification problem. Consequently, state-of-the-art subject-independent keystroke stress detection systems in the literature typically report mean accuracies in the range of \textbf{55\% to 70\%}~\cite{researchgateKeystroke, ijritcc2023}.

\noindent The 69.2\% mean SVM-RBF accuracy achieved in this study therefore places the proposed system at the \textbf{upper boundary of the competitive range for universal, subject-independent models}. Furthermore, three additional factors make this result more significant than a direct numerical comparison with other papers would suggest:

\begin{enumerate}
    \item \textbf{Real-World Ecological Validity:} The majority of high-accuracy papers in this domain collect data in controlled laboratory settings where participants perform uniform, prescribed typing tasks (e.g., copying a fixed paragraph of text). The present study collected data during live, graded university examinations where students worked on open-ended algorithmic programming problems  a far noisier and more ecologically valid environment. Real-world deployments consistently yield lower classification accuracies than lab benchmarks~\cite{researchgateKeystroke}.
    \item \textbf{Rigorous Leakage-Free Validation:} A substantial proportion of the literature applies SMOTE or other synthetic over-sampling techniques to the \textit{full} dataset before splitting it into cross-validation folds. This practice constitutes data leakage, as synthetic minority samples derived from the full dataset contaminate the held-out validation sets, artificially inflating reported accuracy by 5--15\% in typical imbalanced settings~\cite{smoteLeakage}. In the present study, SMOTE was strictly applied \textit{inside} each training fold of the 10$\times$10 Repeated Stratified K-Fold loop, guaranteeing that no synthetic samples from the test split influenced the training process. The 69.2\% reported here is therefore a mathematically honest, leakage-free performance estimate.
    \item \textbf{Extreme Natural Class Imbalance:} The 83\% majority-class distribution in the present dataset represents a far more severe natural imbalance than most benchmark datasets used in the literature, which typically report imbalance ratios closer to 60/40 or 70/30. Achieving 69.2\% in the presence of an 83/17 raw imbalance (reduced to 50/50 through SMOTE) confirms that the system successfully captured genuine behavioral signal rather than defaulting to majority-class prediction.
\end{enumerate}

\subsection{Demographic and Educational Implications}
Deploying this research within a cohort of first-year computer science students during actual graded assessments was essential. The initial transition into advanced computer science curricula is highly demanding, and exam settings create natural, authentic psychological pressure. These findings demonstrate that silent background analytics can accurately detect when a novice programmer is becoming overwhelmed. 

\noindent Looking forward, academic institutions can integrate this non-intrusive tracking model to flag students experiencing high assessment anxiety, enabling automated pacing adjustments, personalized debugging help, or early pedagogical interventions.

\section{Summary}
In this chapter, a zero-intrusion developer anxiety monitoring framework was evaluated using empirical field testing data collected from 67 first-year undergraduate students across four live examination sessions. Analyzing over 11,138 distinct 30-second behavioral observation intervals confirmed statistically significant correlations between passive OS-level interaction features and self-reported psychological stress levels ($r = 0.521$, $p < 0.0001$). Using a leakage-free 10$\times$10 Repeated Stratified K-Fold Cross-Validation protocol with SMOTE applied strictly within training folds, the SVM-RBF classifier achieved a mean accuracy of 69.2\%, placing the system at the upper boundary of the competitive performance range for subject-independent universal keystroke stress detection models in the current literature. The three key differentiators of this study  real-world ecological validity, strict SMOTE leakage prevention, and operation under an 83\% natural class imbalance  establish that the reported accuracy represents a rigorous and reproducible baseline rather than an optimistically inflated benchmark figure.
