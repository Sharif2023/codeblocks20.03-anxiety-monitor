\chapter{Introduction}

This chapter covers the technical background of the project, the motivation behind it, the research objectives, methods, and findings. A foundation is provided for the development of a framework for non-invasive software analysis on an asynchronous basis at the level of the operating system to detect anxiety in novice programmers based on high-dimensional features.

\section{Project Overview}
The project presents an analysis of designing, evaluating, and validating a hardware-independent developer mental well-being model based on a machine learning classifier, customized for Code::Blocks and native console environments. Acting as a light background infrastructure service for collecting data without any invasive hooks or biometric devices, the proposed architectural pattern collects telemetry of low-level human-computer interaction indicators for 30 seconds tumbling windows. In particular, there are 18 separate parameters isolated from the collected telemetry, which include the speed of typing, keyboard latency dispersion, number of switching events of background windows, number of deletions, and compile requests \cite{zhang2023keystroke, epp2015emotion}. Further, the unconditioned interaction data is processed by applying a supervised machine learning classification algorithm, which allows inferring the programmer's current level of distress \cite{acm345697, anxietyPrediction2023}. The use of no biometric equipment, as well as the transfer of active user interface interaction loops to the future work plan of the system, makes the presented solution scalable and analytical framework zero overhead \cite{selfreg2023, arean2025jitai}.

\section{Motivation}
Given that software engineering is a high-level thinking skill, which includes significant levels of logic and abstract reasoning, the unavoidable stressors, such as programming anxiety, mental fatigue, and recursion, are intensified especially in novice learners when facing examination conditions \cite{acm4897123, springer2020}. In the case when the developer faces complicated compilation or semantic obstacles, the generated emotional pressure significantly decreases the working memory capacity, hampers debugging process, and lowers the efficiency of pedagogical learning process \cite{stemedu2025}. \noindent Although IDEs have been created for efficient code generation and syntactical execution, they are not able to perceive and adjust according to the psychological condition of the user, and therefore fail to handle severe panic loops \cite{mdpi5739}. Therefore, there is an increasing necessity to create noninvasive software systems, which will be able to monitor behavioral stressors in real time through native interaction channels \cite{realtimeStress2012}. This will be very important for computer science education and software engineering teams to establish the foundation for emotional intelligent workspaces, which will keep programmer motivation alive and prevent burnout.

\section{Objectives}
The general goal of this work is to implement an intelligent and unobtrusive developer stress tracking system with a supervised machine learning classification engine, using passive software telemetry for classifying distress states of novice users in a real-world setting without adding any additional hardware and performance cost \cite{mdpi389}.

\noindent The concrete goals of this work include:
\begin{itemize}
    \item Designing an asynchronous, system-wide software telemetry collector able to operate in a non-intrusive manner in the background of the workspace without any additional thread, compiler, or user interference delays.
    
    \item Tracking and separating passive behavioral coding features such as high-frequency typing speed changes, keyboard latency, backspacing, and foreground application window descriptions to capture reliable user stress signals \cite{bunia2024keystroke}.
    
    \item Conducting a robust descriptive statistical profiling experiment of your feature space and making a mathematical comparison of interaction geometries of different developers across their ascending levels of stress \cite{anxietyPrediction2023}.
    
    \item Training, tuning, and evaluating several supervised machine learning models including SVM, Random Forest Classifier, and Naive Bayes Classifier models via cross-validation.
    
   \item In order to use the state-of-the-art over-sampling methods (SMOTE) in order to effectively eliminate severe class imbalance due to the natural stress associated with examinations.
        
        \item For providing an empirical benchmark for measuring the classification accuracy of your algorithm against psychological measures and actual surveys.
        
        \item To ensure that there is absolute user data privacy by capturing only non-literal structural frequencies and not alphanumeric string text.
\end{itemize}

\subsection{System Development}
In this stage, the focus is on the construction of a lightweight, decoupled tracking service. By running directly on the operating system level APIs, the telemetry collection skips over version-specific asynchronous monitoring service spaces and avoids compiler priority locks. The technology design takes care of two major sub-modules, which are the asynchronous background data logging module that records high-speed event counts and the serialization module for appending interaction logs in 30-second intervals.

\subsection{Data Collection}
After validating the pipeline for the background infrastructure, data collection is done in a pressure-filled setting to collect authentic responses that correspond to physiological and emotional stresses. In order to do this, the script used in tracking is run on the workstations within universities to observe beginners who solve complex algorithmic questions within limited time frames. There are eighteen different measurements logged by the engine that include number of keystrokes, compilation attempts, switch in focus, and time spent idle.

\subsection{Feature Engineering}
The unconditioned interaction text files generated from active sessions are then processed computationally and represented as multi-dimensional feature spaces. This stage emphasizes the extraction of detailed behavioral predictors from data streams. Some of the operations involve WPM speeds, mapping the variability of keyboard latency to irregular rhythms, backspace delete rates, and counting active focus changes from the main IDE session.

\subsection{Preprocessing and Dataset Balancing}
For the preservation of the classifier from damage due to highly skewed class distributions that tend to prevail (~83\% high stress) during stressed examination conditions, a rigid pre-processing protocol is adopted. The initial intervals with empty text buffers, along with any mechanical outliers, are eliminated, and capped by stringent thresholding procedures. The subsequent training samples are balanced using the SMOTE (Synthetic Minority Over-sampling Technique)..

\subsection{Supervised Model Evaluation}
At the time of the evaluation phase, several machine learning models using supervised learning are subjected to training and cross-validation processes. The entire evaluation phase is executed using a 10$\times$10 Repeated Stratified K-Fold Cross-Validation loop, where SMOTE is applied only within the inner training folds to avoid any form of data leakage. The models are evaluated based on common metrics like Accuracy, Precision, Recall, and F1-score, along with conducting the feature importance test for a Random Forest.

\section{Project Outcome}
Consequently, the main achievement of this study is a proven, software-based machine learning classification model that can detect beginner programming anxiety states with absolutely no need for any physical hardware or process interruption \cite{mdpi5739, rahimi2024ai}. 

\noindent Technical accomplishments and deliverables of this project are as follows:
\begin{itemize}
    \item \textbf{Asynchronous System Telemetry Utility:} An extremely lightweight background monitoring service collecting interaction meta-data on operating system level without imposing any execution delays on the main code editing canvas.
    
    \item \textbf{Multi-Dimensional 18 Features Space Model:} A structured, cleaned and consolidated time series data set which consists of precisely 11,138 unique 30 seconds observation intervals recorded from a group of 67 undergraduate engineering students.
    
    \item \textbf{Descriptive Behavioral Features Profiling Benchmark:} An empiric profiling map indicating distinctive numerical changes in typing speed, latency variance and session fragmentation while a person moves from a state of flow to a state of cognitive stress.
    
    \item \textbf{Optimized Machine Learning Classification Algorithm Back-end:} Comparative analysis schema illustrating the fact that SVM-RBF (Support Vector Machine with Radial Basis Function) has Mean Accuracy of 69.2\% versus 33.3\% random guessing baseline \cite{mdpi389}.
    
    \item \textbf{Predictive Weight Mapping of Features:} Detailed Random Forest feature analysis highlighting keyboard latency variation and compilation success rates as the main predictors of stress among software developers.
        
        \item \textbf{Ethical Privacy Preserving Protocol:} A robust privacy-preserving protocol for data processing to demonstrate that passive affective computing can be used to generate accurate predictions without compromising the confidentiality of the source code.
\end{itemize}
