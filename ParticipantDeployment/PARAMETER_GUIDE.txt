========================================
PARAMETER GUIDE - What Each Column Means
========================================

This document explains all 18 working parameters collected by the system.

========================================
SECTION 1: IDENTIFICATION & TIMING
========================================

1. timestamp
   - What: Date and time of data collection
   - Format: YYYY-MM-DD HH:MM:SS
   - Example: 2026-01-25 13:45:30
   - How collected: System clock
   - Used for: Time series analysis, session duration
   - Research use: Track when anxiety peaks occur

2. session_id
   - What: Unique identifier for this collection session
   - Format: session_HHMMSS
   - Example: session_134530
   - How collected: Generated at start
   - Used for: Grouping data from same session
   - Research use: Separate multiple sessions per participant

3. timestamp_batch
   - What: Time grouping for aggregation
   - Format: YYYY-MM-DD_HHMM (rounded to minute)
   - Example: 2026-01-25_1345
   - How collected: Rounded timestamp
   - Used for: Grouping rows by time period
   - Research use: Analyze behavior over time windows

========================================
SECTION 2: FILE & CONTEXT
========================================

4. file_path
   - What: Name of active file being edited
   - Format: filename.extension
   - Example: main.cpp, utils.h
   - How collected: Extracted from CodeBlocks window title
   - Used for: Track which files user works on
   - Research use: Complexity of files vs anxiety
   - Limitation: Generic name only, not full path

5. language
   - What: Programming language detected
   - Format: Language name or "Unknown"
   - Example: C/C++, Python, Header
   - How collected: From file extension
   - Used for: Language-specific behavior analysis
   - Research use: Does language affect anxiety?

6. window_focused
   - What: Whether CodeBlocks has active focus
   - Format: TRUE or FALSE
   - How collected: Windows API GetForegroundWindow
   - Used for: Distinguish active coding from distractions
   - Research use: Measure focus vs distraction patterns

========================================
SECTION 3: TYPING BEHAVIOR
========================================

7. typing_speed_wpm
   - What: Words per minute typing speed
   - Formula: (keystrokes / 5) / minutes
   - Range: 0-100+ WPM
   - How collected: Calculated from keystroke timing
   - Used for: Measure coding fluency
   - Research use: LOW speed = struggling, HIGH speed = flow state
   - Pattern: Drops when debugging, rises when confident

8. latency_variance_ms
   - What: Standard deviation of inter-keystroke delays
   - Unit: Milliseconds
   - Range: 0-500+ ms
   - How collected: Statistics.stdev() on key delays
   - Used for: Detect typing rhythm irregularity
   - Research use: HIGH variance = stress/anxiety indicator
   - Pattern: Increases under pressure or confusion

9. backspace_rate
   - What: Percentage of keystrokes that are backspace
   - Formula: (backspaces / total_keystrokes) * 100
   - Range: 0-50%
   - How collected: Backspace counting vs total keys
   - Used for: Measure error correction behavior
   - Research use: HIGH rate = making mistakes, uncertainty
   - Pattern: Spikes during debugging or trial-and-error

10. keystrokes_total
    - What: Total number of keys pressed this session
    - Range: 0 to thousands
    - How collected: Incremental counter
    - Used for: Measure overall activity level
    - Research use: Productivity metric, work intensity

========================================
SECTION 4: PAUSE & IDLE BEHAVIOR
========================================

11. pause_ratio
    - What: Proportion of time spent in long pauses (>30s)
    - Range: 0.0 to 1.0
    - Formula: (total_pause_time / session_time)
    - How collected: Track pauses >30 seconds
    - Used for: Measure thinking/planning time
    - Research use: HIGH ratio = contemplation or blocking
    - Pattern: Normal 0.1-0.3, high 0.5+

12. idle_ratio
    - What: Time since last keystroke (normalized to 30s)
    - Range: 0.0 to 1.0
    - Formula: min(seconds_idle / 30, 1.0)
    - How collected: Time since last key press
    - Used for: Detect current idle state
    - Research use: Are they thinking or stuck?
    - Pattern: Resets every keystroke

13. session_fragmentation
    - What: How often work is interrupted by long pauses
    - Range: 0.0 to 1.0
    - Formula: (pauses_per_10min / 5.0)
    - How collected: Count pauses >30s, normalize
    - Used for: Measure work continuity
    - Research use: HIGH = context switching, interruptions
    - Pattern: <0.2 good flow, >0.5 fragmented work

========================================
SECTION 5: COMPILE BEHAVIOR
========================================

14. compile_attempts
    - What: Number of times code was compiled
    - Range: 0 to hundreds
    - How collected: Detect gcc.exe process starts
    - Used for: Measure compile frequency
    - Research use: Many compiles = trial-and-error approach
    - Pattern: Increases during debugging
    - Limitation: May miss fast compiles

15. compile_success_rate
    - What: Estimated percentage of successful compiles
    - Range: 50-90%
    - Formula: Based on backspace rate heuristic
    - How collected: Estimated from error correction
    - Used for: Approximate compile success
    - Research use: Success rate vs anxiety correlation
    - Note: ESTIMATED, not exact (requires plugin for exact)

========================================
SECTION 6: FOCUS & ATTENTION
========================================

16. focus_switches
    - What: Number of times window focus changed
    - Range: 0 to hundreds
    - How collected: Count window title changes
    - Used for: Measure task switching
    - Research use: HIGH = distraction, multitasking
    - Pattern: Low (0-5) = focused, High (20+) = scattered

========================================
SECTION 7: ANXIETY INDICATORS
========================================

17. anxiety_score
    - What: Calculated stress/anxiety level
    - Range: 0-100
    - Formula: Weighted sum of behavioral indicators
    - Components:
      * High latency variance (+15)
      * High backspace rate (+25)
      * Low typing speed (+10)
      * High pause ratio (+20)
      * Many compiles (+15)
      * High fragmentation (+15)
    - How collected: Algorithm from behavioral metrics
    - Used for: Primary research metric
    - Research use: Main dependent variable
    - Validated by: Lau (2018), Yu et al. (2025) studies

18. risk_level
    - What: Categorical anxiety level
    - Values: LOW, MODERATE, HIGH, CRITICAL
    - Thresholds:
      * LOW: 0-30 (normal coding)
      * MODERATE: 31-60 (elevated stress)
      * HIGH: 61-80 (significant anxiety)
      * CRITICAL: 81-100 (acute distress)
    - How collected: From anxiety_score
    - Used for: Quick visual assessment
    - Research use: Categorical analysis, intervention triggers

========================================
HOW DATA IS COLLECTED (TECHNICAL)
========================================

KEYSTROKE MONITORING:
- Windows API: GetAsyncKeyState()
- Polling rate: Every 10 milliseconds
- Keys tracked: A-Z, 0-9, Space, Backspace
- Method: Background thread monitors key state changes

WINDOW DETECTION:
- Windows API: GetForegroundWindow(), GetWindowTextW()
- Check rate: When data row collected (every 30s)
- Detection: Searches for "CodeBlocks" in window title

COMPILE DETECTION:
- Windows: tasklist command
- Search: gcc.exe process
- Cooldown: 3 seconds between counts (avoid duplicates)

TIMING CALCULATIONS:
- Inter-key delay: Time between consecutive keystrokes
- Variance: Standard deviation of delays
- WPM: Word estimation (5 keys = 1 word)

========================================
DATA QUALITY NOTES
========================================

HIGH QUALITY (>95% accuracy):
- timestamp, session_id
- keystrokes_total, backspace_rate
- typing_speed_wpm
- window_focused

GOOD QUALITY (80-95% accuracy):
- latency_variance_ms
- pause_ratio, idle_ratio
- session_fragmentation
- focus_switches
- anxiety_score

ESTIMATED (50-80% accuracy):
- compile_attempts (may miss fast compiles)
- compile_success_rate (heuristic-based)
- file_path (generic names only)

========================================
LIMITATIONS TO CONSIDER
========================================

CANNOT DETECT:
- Specific compiler errors
- Undo/redo commands
- Code quality or correctness
- Exact file paths or project structure

POTENTIAL CONFOUNDS:
- Keyboard breaks (drinking, thinking)
- Copy-paste from external sources
- Reading documentation vs coding
- Pair programming or code review

MITIGATION:
- Long sessions reduce noise
- Multiple participants average out variance
- Combine with self-report data
- Use window_focused to filter active coding
